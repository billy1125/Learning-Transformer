# 高中生版：Transformer 注意力機制（簡單版）

我們不從數學公式出發，而是用 **「圖書館找書」和「螢光筆畫重點」** 來理解。

## 1. 什麼是 Embedding？（把文字變數字）

電腦不懂什麼是「貓」。我們要給貓一個「座標」。

例子：假設有兩個維度：[毛茸茸程度, 體型大小]。

- 「貓」可能是 [0.9, 0.2]
- 「老虎」可能是 [0.8, 0.9]

因為這兩個點在空間中很近，電腦就知道它們語意相關。這就是 Embedding。

## 2. Q、K、V 是什麼？（圖書館類比）

想像你在圖書館（這是一整個序列的文字）：

- Query (Q) — 你的搜尋關鍵字：你手裡拿著一張紙，寫著「我想找關於『吃』的動作」。
- Key (K) — 書架上的標籤：每本書的書背都有標籤。有的寫「食物」、有的寫「運動」。
- Value (V) — 書裡的內容：當標籤（K）跟你手裡的紙（Q）配對成功，你就把這本書（V）翻開來讀。

核心概念：我們把每個字都變出 Q、K、V 三種身分。當前的字（Q）會去跟全場所有的字（K）比對，看誰跟自己最配。

## 3. 注意力分數：誰才是重點？

我們用「內積」來計算相似度。在高中數學裡，向量內積越大，代表方向越接近。

- 當 Q(吃) 遇到 K(魚)，內積很大 → 分數高。
- 當 Q(吃) 遇到 K(昨天)，內積很小 → 分數低。

## 4. Softmax：分配「關注百分比」

我們會把分數轉成百分比（加總等於 100%）。

比如：「吃」這個字，分配了 80% 的注意力給「魚」，15% 給「貓」，5% 給「昨天」。

這就像是你讀書時，用螢光筆把「魚」畫得最粗。

## 5. 為什麼要除以 d

> 防止數據「爆炸」

高中數學學過，如果向量長度（維度）增加，內積數值會變得非常大。

數值太大會讓 Softmax 變得很極端（只有一個 100%，其他都是 0%），這樣模型就沒辦法透過微小的調整來學習。除以這個數字，是為了讓分數「冷靜下來」。

# 🌟 Transformer 核心概念

> Transformer 就是讓句子裡的每個字，都根據「搜尋需求(Q)」去對比其他字的「標籤(K)」，算出「重要性(A)」，最後把「有用的資訊(V)」打包帶走。

# 📖 給高中生的挑戰小筆記

如果你想挑戰原始教材，只要記住這三個矩陣的形狀（Shape）：

- X (T×d)：T 是有幾個字，d 是每個字有多長。
- A (T×T)：這是一個「誰對誰感興趣」的表格。
- C (T×d)：最後產出的結果，每個字都吸收了全場的精華。
